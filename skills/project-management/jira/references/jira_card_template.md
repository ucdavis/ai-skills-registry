# JIRA Card Template

Use this structure for every generated markdown card. Replace bracketed text with concrete details.

```markdown
# [Card Title]

## Summary

One paragraph describing the problem/opportunity, affected users, and why the work matters.

## Context

- Current behavior / evidence (logs, metrics, screenshots)
- Linked docs (PRD, dashboards, previous tickets)
- Constraints or assumptions

## Requirements / Acceptance Criteria

1. [Clear, testable behavior]
2. [Edge cases and error states]
3. [Performance / accessibility standards]

## Deliverables

- [Code changes, configs, migrations]
- [Documentation / runbooks]
- [QA artifacts or sign-off]

## Definition of Done

- [Deployment + verification checklist]
- [Observability / alert updates]
- [Retro or stakeholder approval steps]
```

Add additional sections (e.g., Rollout Plan, Analytics, Open Questions) when the project scope requires them.
